# Testing Portfolio

This folder contains software testing artefacts, automation practice, accessibility defect examples, and supporting evidence for my portfolio.

The focus is on documenting practical testing work clearly: what was tested, how it was tested, what evidence was captured, and how the artefact demonstrates software quality practice.

---

## Testing areas

| Area                                                          | Description                                                                                                        |
| ------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| [`test-scenarios`](./test-scenarios/)                         | High-level testing scenarios that describe what needs to be checked and why.                                       |
| [`test-cases`](./test-cases/)                                 | More detailed test cases with steps, expected results, and supporting notes.                                       |
| [`defect-reports`](./defect-reports/)                         | Bug reports with reproduction steps, expected results, actual results, severity, priority, and evidence.           |
| [`api-testing-postman`](./api-testing-postman/)               | API testing artefacts using Postman, including requests, assertions, environments, and CRUD workflows.             |
| [`performance-testing-jmeter`](./performance-testing-jmeter/) | JMeter practice and performance testing notes.                                                                     |
| [`ui-automation-selenium`](./ui-automation-selenium/)         | Selenium WebDriver practice and UI automation examples.                                                            |
| [`ui-automation-katalon`](./ui-automation-katalon/)           | Katalon Studio practice and UI automation examples.                                                                |
| [`sql-testing`](./sql-testing/)                               | SQL queries and data checks used to support testing evidence.                                                      |
| [`accessibility-defects`](./accessibility-defects/)           | Accessibility-focused defect examples connected to WCAG, automated findings, manual review, and remediation notes. |
| [`screenshots`](./screenshots/)                               | Screenshot evidence used to support testing notes, defect reports, and portfolio documentation.                    |

---

## Purpose of this folder

This folder supports my portfolio direction as an aspiring **Test Automation Engineer** and **Digital Accessibility Advocate**.

It is intended to show evidence across:

* manual testing documentation
* test scenario design
* test case writing
* defect reporting
* API testing
* UI automation practice
* SQL-based checks
* performance testing basics
* accessibility-aware testing
* screenshot-based evidence

---

## Artefact approach

Where possible, artefacts will be documented with enough context to make them understandable outside the original course or project environment.

This may include:

* what the artefact demonstrates
* the tool or method used
* the system or feature under test
* relevant screenshots
* limitations or assumptions
* links to related project work
